    二叉树的最大深度¶

    题目描述¶

    给定一个二叉树，找出其最大深度。

    二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

    说明: 叶子节点是指没有子节点的节点。

    解题思路¶

    这个问题可以通过两种方法解决： 1. 递归解法：使用DFS（深度优先搜索）计算深度 2. 迭代解法：使用BFS（广度优先搜索）或层序遍历计算深度

    最简单的方法（递归DFS）¶

    使用递归计算左子树和右子树的最大深度，取较大值加1。

    # 定义二叉树节点类
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def max_depth_brute(root):
    # 基本情况：空节点
    if not root:
        return 0

    # 递归计算左子树的最大深度
    left_depth = max_depth_brute(root.left)
    # 递归计算右子树的最大深度
    right_depth = max_depth_brute(root.right)

    # 返回较大的深度加1（加1是因为要包含当前节点）
    return max(left_depth, right_depth) + 1

    最优方案（迭代BFS/层序遍历）¶

    使用队列进行层序遍历，记录层数。

    from collections import deque

def max_depth_optimal(root):
    # 特殊情况处理
    if not root:
        return 0

    # 使用队列进行层序遍历
    queue = deque([root])
    depth = 0

    # 当队列不为空时，继续遍历
    while queue:
        # 获取当前层的节点数量
        level_size = len(queue)
        # 层数加1
        depth += 1

        # 处理当前层的所有节点
        for _ in range(level_size):
            node = queue.popleft()
            # 将子节点加入队列
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)

    return depth

    迭代DFS解法（可选）¶

    使用栈模拟递归过程：

    def max_depth_dfs_iterative(root):
    if not root:
        return 0

    stack = [(root, 1)]  # 栈中的元素是(节点, 深度)
    max_depth = 0

    while stack:
        node, current_depth = stack.pop()
        max_depth = max(max_depth, current_depth)

        # 先将右子节点入栈，再将左子节点入栈，这样出栈顺序是先左后右
        if node.right:
            stack.append((node.right, current_depth + 1))
        if node.left:
            stack.append((node.left, current_depth + 1))

    return max_depth

    复杂度分析¶

    • 递归DFS解法：
    • 时间复杂度：O(n)，其中 n 是二叉树中的节点数，每个节点都会被访问一次

    • 空间复杂度：O(h)，其中 h 是树的高度，最坏情况下（树完全不平衡）为 O(n)，最好情况下（树完全平衡）为 O(log n)

    • 迭代BFS解法：

    • 时间复杂度：O(n)，每个节点都会被访问一次
    • 空间复杂度：O(w)，其中 w 是树的最大宽度，最坏情况下为 O(n/2) = O(n)

    递归DFS解法虽然实现简单，但在树很深的情况下可能会有栈溢出的风险。迭代解法可以避免这个问题，两种方法的时间复杂度相同，但在空间使用上各有特点。
