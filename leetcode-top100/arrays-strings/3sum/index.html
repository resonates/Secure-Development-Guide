    三数之和¶

    题目描述¶

    给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请

    你返回所有和为 0 且不重复的三元组。

    注意： 答案中不可以包含重复的三元组。

    示例 1：¶

    输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

    示例 2：¶

    输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。

    示例 3：¶

    输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。

    提示：¶

    • 3 <= nums.length <= 3000
    • -10⁵ <= nums[i] <= 10⁵

    解题思路¶

    这个问题可以通过两种方法解决： 1. 暴力解法：三重循环遍历所有可能的三元组 2. 排序+双指针：先排序，然后使用双指针查找

    最简单的方法（暴力解法）¶

    使用三重循环遍历所有可能的三元组，检查它们的和是否为 0，并去重。

    def three_sum_brute(nums):
    if len(nums) < 3:
        return []

    result = set()
    # 遍历所有可能的三元组
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            for k in range(j + 1, len(nums)):
                # 检查三数之和是否为0
                if nums[i] + nums[j] + nums[k] == 0:
                    # 将三元组排序后存入集合，用于去重
                    triplet = tuple(sorted([nums[i], nums[j], nums[k]]))
                    result.add(triplet)

    # 将集合转换为列表
    return [list(triplet) for triplet in result]

    最优方案（排序+双指针法）¶

    先对数组进行排序，然后固定一个元素，使用双指针查找另外两个元素，使得它们的和为目标值的相反数。

    def three_sum_optimal(nums):
    if len(nums) < 3:
        return []

    # 排序数组
    nums.sort()
    result = []

    # 遍历每个元素作为第一个元素
    for i in range(len(nums)):
        # 如果当前元素大于0，由于数组已排序，后面的元素都大于0，不可能和为0
        if nums[i] > 0:
            break

        # 跳过重复的第一个元素
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        # 设置左右指针
        left = i + 1
        right = len(nums) - 1

        # 使用双指针查找另外两个元素
        while left < right:
            total = nums[i] + nums[left] + nums[right]

            if total == 0:
                # 找到符合条件的三元组
                result.append([nums[i], nums[left], nums[right]])

                # 跳过重复的左指针元素
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                # 跳过重复的右指针元素
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1

                # 移动指针
                left += 1
                right -= 1
            elif total < 0:
                # 总和太小，移动左指针
                left += 1
            else:
                # 总和太大，移动右指针
                right -= 1

    return result

    复杂度分析¶

    • 暴力解法：
    • 时间复杂度：O(n³)，其中 n 是数组长度

    • 空间复杂度：O(n)，用于存储结果

    • 排序+双指针解法：

    • 时间复杂度：O(n²)，排序需要 O(nlogn)，双指针查找需要 O(n²)
    • 空间复杂度：O(1)，除了存储结果外不需要额外空间

    排序+双指针解法通过排序和双指针技术，将时间复杂度从 O(n³) 降低到了 O(n²)，同时避免了使用集合去重，大大提高了效率。
