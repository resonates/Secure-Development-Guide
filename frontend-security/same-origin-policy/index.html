    同源策略 (Same-Origin Policy)¶

    什么是同源策略？¶

    同源策略是浏览器的一个重要安全机制，它限制了一个源（origin）的文档或者它加载的脚本如何与另一个源的资源进行交互。这是为了防止恶意网站窃取用户数据或进行未授权的操作。

    同源的定义¶

    两个URL被认为是同源，当且仅当它们满足以下三个条件完全相同：

    1. 协议(protocol) 相同
    2. 域名(domain) 相同
    3. 端口(port) 相同（如果指定了端口）

    同源检测示例¶

    以下是一些同源检测的示例：

    原始URL 比较URL 是否同源 原因
    http://example.com/page1 http://example.com/page2 是 协议、域名、端口都相同
    http://example.com:8080 http://example.com:8080/data 是 协议、域名、端口都相同
    http://example.com https://example.com 否 协议不同（http vs https）
    http://example.com http://www.example.com 否 域名不同
    http://example.com http://api.example.com 否 域名不同（子域名不同）
    http://example.com:80 http://example.com:8080 否 端口不同

    同源策略的限制范围¶

    同源策略主要限制以下几个方面：

    1. DOM 访问¶

    一个源的JavaScript不能直接访问另一个源页面的DOM。

    // 以下代码如果在不同源的页面中执行，会抛出安全错误
const iframe = document.getElementById('myIframe');
const otherDoc = iframe.contentDocument; // 跨源时会报错

    2. XMLHttpRequest 和 Fetch API¶

    默认情况下，这些API只能发起同源请求，不能向其他源发送请求。

    // 跨源请求默认会被阻止
fetch('https://different-origin.com/data')
  .then(response => response.json())
  .catch(error => console.error('跨域请求错误:', error));

    3. Cookie、LocalStorage 和 IndexedDB¶

    这些存储机制受到同源策略的保护，一个源不能访问另一个源的数据。

    // 只能访问当前源的localStorage数据
localStorage.setItem('user', 'data'); // 只能在同一源读取

    4. 其他资源访问限制¶

    如CSS、JavaScript文件虽然可以跨源加载，但受限于同源策略的其他规则。

    同源策略的例外¶

    1. 允许跨源嵌入的资源¶

    以下资源默认允许跨源嵌入：

    • <script> 标签加载的JavaScript
    • <link> 标签加载的CSS
    • <img> 标签加载的图片
    • <video> 和 <audio> 标签加载的媒体文件
    • <iframe> 嵌入的任何资源
    • <object>, <embed>, <applet> 嵌入的插件

    2. 跨源资源共享 (CORS)¶

    通过服务器配置特定的HTTP头，允许受控的跨源请求。

    3. JSONP (JSON with Padding)¶

    利用<script>标签不受同源策略限制的特性，实现跨源数据获取。

    同源策略的重要性¶

    同源策略是Web安全的基石，它提供了以下保护：

    1. 防止数据窃取：防止恶意网站读取另一个网站的敏感数据
    2. 保护用户凭证：防止跨站会话劫持
    3. 维护网站独立性：确保每个网站在自己的安全沙箱中运行
    4. 防止恶意操作：限制未授权的跨站操作

    同源策略的绕过风险¶

    尽管同源策略提供了良好的保护，但仍存在一些绕过风险：

    • XSS攻击：如果网站存在XSS漏洞，攻击者可以注入脚本在同一源内执行，从而绕过同源策略
    • CSRF攻击：利用用户已有的认证状态发起跨站请求
    • iframe嵌套攻击：通过某些技巧绕过iframe的同源限制

    如何安全处理跨源交互¶

    1. 使用CORS进行合法跨域请求
    2. 实施适当的内容安全策略(CSP)
    3. 正确配置Cookie属性(SameSite, HttpOnly, Secure)
    4. 对所有用户输入进行严格验证和转义
    5. 使用postMessage API进行安全的跨源通信

    总结¶

    同源策略是浏览器安全的基础机制，它通过限制不同源之间的资源访问，有效防范了各种恶意攻击。了解同源策略的工作原理和限制，对于构建安全的Web应用至关重要。在需要进行跨源交互时，应当采用安全的机制如CORS或postMessage，并确保正确实施各种安全最佳实践。
